# Codex Monitor v0.2.0

Codex Monitor v0.2.0 makes long-running monitoring reliable and gives the dashboard a clear operational health state.

## Highlights

- Token totals now represent usage since the monitor launched and no longer disappear when agents become inactive.
- Large Codex session logs are read backward from their newest records, avoiding the refresh stalls seen with multi-megabyte JSONL files.
- The dashboard shows active workspaces and agents, uptime, last token refresh, and `OK` or sanitized warning status.
- Filesystem watcher overflows are visible instead of silently producing incomplete activity counts.
- New SVG, PNG, and Windows ICO branding accompanies refreshed privacy-safe screenshots.

## Fixes and reliability

- Preserve accumulated token deltas through session inactivity, resumed activity, and upstream counter rollback.
- Ignore partially written final JSONL records while accepting the newest complete token record.
- Clear stale terminal rows before in-place redraws.
- Retain the five-second session recovery scan alongside event-driven refresh.

## Upgrade

Download and extract `codex-monitor-v0.2.0.zip`, then run `Start-Codex-Monitor.cmd`. No configuration or persistent-data migration is required.

Token counters intentionally restart at zero for each new monitor process because they measure usage since launch.

## Privacy

No telemetry is added. The monitor continues to read only session ID, working directory, modification time, and token-count records. Health warnings contain sanitized counts, never record bodies or source content.

## Compatibility and rollback

Codex Monitor supports Windows PowerShell 5.1 and PowerShell 7. To roll back, replace the v0.2.0 files with v0.1.1; no stored data requires migration.
